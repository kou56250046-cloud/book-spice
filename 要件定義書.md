## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| サイト名 | **人生のスパイス** |
| サイトキャッチ | 人生にひと匙の刺激を。 |
| 目的 | 日本語のベストセラー本をジャンル別にすぐ探せるようにする |
| 最終ゴール | 「人生を変えるほどインパクトのある本」をすぐ見つけられる場所をつくる |
| 利用者 | 自分自身（個人用） |
| 公開方法 | GitHub Pages による公開リンク |

### コンセプト

5つのスパイスジャンルで人生に刺激を与える本を探せる、落ち着いた書店・図書館風の読書ガイドサイト。データ更新は毎週土曜に完全自動。コメントはオーナー自身が書いた言葉で個性を出す。

---

## 2. 前提・制約条件

- **完全無料**で構築・運用する（ランニングコスト 0 円）
- **個人用**（ログイン機能・複数ユーザー管理は不要）
- **日本語の本**をメインに扱う
- **毎週土曜 朝7:00 JST** に週次自動更新
- ホスティングは **GitHub Pages**、自動更新は **GitHub Actions** を使う
- データは **楽天ブックス系API（無料）** から取得する
  - 利用には楽天アプリID（無料・3分程度）の取得が必要
  - **クレジット表記が必須**（楽天ウェブサービスの規約）
- **Claude API は使用しない**（完全無料を優先）

| No | 機能 | 内容 | 保存先 |
|----|------|------|--------|
| F-07 | スパイス度表示 | 各書籍の「人生へのインパクト度」を種粒メーターで表示 | 算出ロジック（API値ベース） |
| F-08 | お気に入り・読了チェック保存 | 「お気に入り」「読了」を記録・絞り込み | ブラウザ内（localStorage） |

| F-10 | 一言レビュー・名言メモ | 自分用の感想・心に残った言葉を記録 | ブラウザ内（localStorage） |
| F-11 | 並べ替え | スパイス度順／ランキング順／タイトル順 | クライアント側処理 |

> **補足：** F-08／F-10 のユーザーデータはブラウザの localStorage に端末内保存。別の端末とは共有されない仕様。個人用のため問題なしと確認済み。

### 4.3 コメント方針（確定）

| 項目 | 方針 |
|------|------|
| 生成方法 | **オーナー自身が手書き**（Claude API は使わない） |
| 項目 | ①おすすめの理由（1〜2文）　②人生にスパイスを与える視点（1文） |
| 更新タイミング | 定番本は一度書いたら使い回し。新着本は気が向いたときに追加 |
| コメント未記入時 | 「コメントなし」として表示（エラーにならない） |
| 管理方法 | リポジトリ内の `comments.json` をオーナーが直接編集 |

> **採用理由：** AI生成より自分の言葉の方がサイトの個性になる。手間は定番本の初回のみ。

## 6. システム構成・技術スタック

```
[GitHub Actions（毎週土曜 22:00 UTC = 土曜 07:00 JST）]
        │  ① 楽天ブックスAPIを呼び出し
        ▼
[楽天ブックス系API（無料）]
        │  ② 書籍データ（ランキング・書影・あらすじ等）を取得
        ▼
[ビルドスクリプト（Node.js）]
        │  ③ books.json を生成（comments.json とマージ）
        ▼
[GitHubリポジトリにcommit]
        │  ④ 自動デプロイ
        ▼
[GitHub Pages（静的サイト）] ──→ 公開リンクで閲覧
```

| レイヤー | 採用技術 |
|----------|----------|
| フロントエンド | HTML / CSS / Vanilla JavaScript（ライブラリ最小限） |
| フォント | Shippori Mincho B1（明朝・見出し）／ Noto Sans JP（本文） |
| データ形式 | JSON（books.json・comments.json をリポジトリ内に保存） |
| ホスティング | GitHub Pages |
| 定期実行・自動化 | GitHub Actions（cron: 毎週土曜 22:00 UTC） |
| データソース | 楽天ブックス書籍検索API／ランキングAPI |
| 秘密情報管理 | 楽天アプリIDは GitHub Actions Secrets に格納（公開しない） |

## 8. スパイス度の算出方針

API から取れる客観指標で「人生へのインパクト」を近似する。

```
スパイス度（1〜5）=
  レビュー平均評価（5点満点）× 0.5
  + レビュー件数（対数スケールで正規化）× 0.3
  + ランキング上位補正 × 0.2
```

- 「評価が高く、長く多くの人に支持されている本」が高スコアになるよう重み付け
- フェーズ2では定番名著に**手動スパイス度**を設定し、売上に依存しない評価軸を追加

## 10. デザイン要件（確定）

- 方向性：**落ち着いた書店・図書館風 × スパイスのコンセプト**
- 背景：和紙色（#ECE3D0）＋ 微細なドットパターン
- 見出し：Shippori Mincho B1（明朝体）
- ジャンルは各スパイスの色で色分け（5色）
- ★の代わりに**菱形の種粒メーター**でスパイス度を表現（サイト固有の署名要素）
- 余白を広めにとった書籍カードレイアウト
- モバイルファースト・レスポンシブ対応

---

## 11. 更新フロー（週次自動化・確定）

1. 毎週土曜 22:00 UTC（= 日本時間 土曜 07:00）に GitHub Actions が起動
2. 楽天ブックスAPIから各ジャンルの最新ランキング上位20冊を取得
3. `comments.json`（オーナー手書きコメント）とマージして `books.json` を生成
4. リポジトリへ自動 commit
5. GitHub Pages が自動で再デプロイ → 最新版が公開

## 12. コンプライアンス・利用規約

- 楽天ウェブサービスの**利用規約を遵守**する
- フッターに**楽天クレジット表記**を必ず掲載（規約上必須）
- 楽天アプリIDは GitHub Secrets に格納し、公開リポジトリに露出させない
- アフィリエイトは**不使用**（完全無料・個人用のため）
